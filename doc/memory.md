## 1. 文档目的
本文档作为嵌入式固件开发的核心参考，明确Bootloader（FBL）和应用程序（APP）在MCU Flash及专用寄存器中的地址分配、长度、用途、校验规则，确保FBL与APP地址无冲突、启动逻辑可靠，适用于开发、编译、烧录、调试全流程。

## 2. 核心术语定义
| 术语        | 详细解释                                                                 |
|-------------|--------------------------------------------------------------------------|
| FBL         | Flash Bootloader，基于片上Flash的引导程序，负责APP的有效性校验和启动加载  |
| APP         | Application，嵌入式应用程序主体，包含业务逻辑、外设驱动等核心功能          |
| Flash       | MCU片上闪存，非易失性存储介质，用于固化FBL、APP代码及配置数据             |
| 有效标志位  | 固定值校验地址，FBL通过读取该地址值判断APP是否合法、可执行                |
| 交互寄存器  | MCU专用寄存器，用于FBL与APP之间传递启动状态（如是否停留FBL、是否升级）    |
| 隔离区      | FBL与APP之间的预留空间，用于避免地址重叠，提升固件兼容性                  |

## 3. 内存分配核心总表
| 存储类型       | 区域名称                | 地址范围                | 等效区间（K） | 长度   | 核心用途                          | 格式/校验要求                          |
|----------------|-------------------------|-------------------------|----------------|--------|-----------------------------------|---------------------------------------|
| **Flash存储**  | FBL核心代码区           | 0x08000000 - 0x08007FFF | 0K - 32K       | 32K    | 存放FBL引导逻辑、硬件驱动、校验代码 | 只读，编译链接起始地址0x08000000，长度≤32K |
|                | FBL版本号存储地址       | 0x00007FF0,4              | -              | 4Byte  | 存储FBL版本号（如V1.0）           | ASCII格式，4字节固定长度，支持运行时读取 |
|                | FBL-APP隔离预留区       | 0x08008000 - 0x08009FFF | 32K - 40K      | 8K     | 避免FBL与APP地址重叠，提升兼容性   | 暂未使用，可扩展为配置存储区            |
|                | APP整体区域             | 0x0800A000 - 0x0801FFFF | 40K - 128K     | 88K    | APP总存储空间（含前导区+核心代码区） | 可读写，APP编译链接主区间               |
|                | APP有效标志地址         | 0x080A0000 - 0x080BFFF  | 40K - 48K      | 4Byte  | APP有效性校验位                   | 有效值固定为0x5A5A5A5A，非该值则APP无效 |
|                | APP核心代码区           | 0x0800C000 - 0x0801FFFF | 48K - 128K     | 80K    | 存放APP业务逻辑、应用核心代码      | 只读，占APP总空间90%以上                |
|                | APP版本号存储地址       | 0x0001FFF0,4              | -              | 4Byte  | 存储APP版本号（如V2.1）           | ASCII格式，与FBL版本号格式保持一致      |
